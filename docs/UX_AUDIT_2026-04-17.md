## Fixes landed during the audit (all in this session's commits)

### 1. Engine disconnection — 3 compounding bugs

**Symptom:** Banner says "Engine disconnected" and never resolves. Clicking "Reconnect" appears to do nothing.

**Root causes:**

a. **Socket wait too short.** `sidecar.rs` [spawn_with_policy](desktop-app/src-tauri/src/sidecar.rs) waited only 5s for the Unix socket after spawn. `npx tsx src/daemon/start.ts` cold-start takes 15–25s because tsx JIT-transforms ~200 TS modules, initializes the runtime (providers/memory/hooks/sandbox), then binds the socket. The 5s deadline ran out → "standalone mode".

b. **Watchdog armed only on success.** In `lib.rs` setup, the watchdog (`state.sidecar.start_watchdog()`) was called inside `if state.sidecar.is_running()`. When (a) timed out, `is_running()` returned false → watchdog never started → **no automatic retry ever**, even though the daemon was completing startup fine in the background.

c. **Stale socket ghost.** After a SIGKILL'd daemon, `~/.wotann/kairos.sock` would be gone from the filesystem but the kernel still had an abstract socket registered. Confirmed via `netstat | grep wotann` showing a stream socket at the same path `ls` said was "No such file or directory". Subsequent connect()s failed.

**Fix:**
- Socket wait 5s → 30s.
- Watchdog now starts unconditionally.
- Watchdog polls every 5s (was 15s), tracks healthy↔unhealthy transitions, auto-respawns with exponential back-off up to 60s on persistent failures.
- On every spawn attempt, stale socket file is removed first (covers the zombie case).
- `AppShell`'s Reconnect button now invokes `restart_engine` (which actually respawns) before re-querying; previously it only re-queried.

### 2. Provider/model picker lied — "gpt selected but only gemma available"

**Symptom:** Header pill shows a model (e.g. gpt-5) that isn't in the dropdown. Clicking the dropdown reveals a completely different provider list (e.g. only Ollama's gemma).

**Root causes:**

a. **Hardcoded `model: "auto"` in Rust.** `commands.rs::get_status` [line 271 pre-fix](desktop-app/src-tauri/src/commands.rs) returned `model: "auto".into()` regardless of what KAIROS reported. Frontend state snapshot then had provider from KAIROS but model from localStorage (or the hardcoded default), creating an impossible pair.

b. **Ollama-only fallback.** When KAIROS is unavailable, `hardcoded_providers()` only probed `http://localhost:11434/api/tags` — so the fallback list was just Ollama regardless of whether the user had OPENAI_API_KEY, ANTHROPIC_API_KEY, etc. set.

c. **No reconciliation on init.** `initializeFromEngine` blindly restored `wotann-selected-provider` / `wotann-selected-model` from localStorage without verifying the pair still existed in `providers[]`. Stale selection → stuck header.

d. **setProvider accepted invalid pairs silently.** `setProvider("openai", "gpt-5")` would happily set state even if OpenAI wasn't in `providers[]`.

**Fix:**
- `get_status` now returns the real model from KAIROS (prefers `result.activeModel`, falls back to `result.model`, falls back to local state — never hardcoded "auto").
- `hardcoded_providers()` probes all 6 common providers (Ollama local + ANTHROPIC / OPENAI / GEMINI / GROQ / CEREBRAS env vars).
- `initializeFromEngine` reconciles: saved localStorage pair → validate against live providers[] → if invalid, snap to first enabled and surface a notification "Provider no longer available".
- `setProvider` validates before accepting — invalid pairs push an error notification.
- `ModelPicker` has a new reconciliation effect: when providers[] updates and current selection becomes invalid, auto-snap to first enabled.

## Top items for session 9

1. **Editor view ErrorBoundary** — CHAT-1 is the biggest user-facing hole. Lazy import chain on Monaco or one of its dependents is throwing. Reproduce via Cmd+2 on a fresh launch.
2. **"Continue without pairing" iOS flow** — IOS-DEEP-1. Surface OnDeviceAI / Settings / Diagnostics without requiring a Mac on the other end.

4. **Providers empty-state CTA** — SET-PROV-1 "Run discovery to populate the list" needs a visible button, not just text.
5. **More Tauri UI adoptions of ValknutSpinner** — `grep animate-spin` still finds ~40 sites; one codemod sweep replaces them all.
