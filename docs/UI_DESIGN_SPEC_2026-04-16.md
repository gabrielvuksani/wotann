## 1. Design thesis

*"Built for those who seek wisdom from the well, not another polished surface."*

Every AI IDE competitor converges on: dark-chrome sidebar, monospace
font, purple accent, translucent panels. **WOTANN is not a "faster
Cursor."** It is a **workshop**, a **council chamber**, and a **proving
ground** — a place where an operator summons intelligence, debates with
it, and leaves with a sealed, auditable proof.

The design language fuses:
- **Norse craft** (forged metal, carved stone, aged vellum, knotwork)
- **Frontier-AI precision** (typographic rigor, scientific instrumentation, surgical motion)

Never cosplay, never kitsch. Think: Dieter Rams' Braun catalog meets the
Lewis Chessmen meets Bloomberg Terminal.

## 4. Seven signature interactions

### 4.1 The Runering (memory-save feedback)

When `mem_save` fires, an 18px Elder Futhark glyph appears at the top-right of the active panel (Ansuz for decisions, Raidho for patterns, Kenaz for discoveries, Naudhiz for blockers). A 1px gold stroke traces a full 360° circle around it in 480ms `cubic-bezier(0.65, 0, 0.35, 1)`. The circle completes, the glyph pulses once (scale 1→1.08→1, opacity 1→0.6 over 280ms), then both fade over 600ms.

### 4.2 Huginn & Muninn — Twin-Raven Split Pane

Keyboard `⌘⇧2` splits the conversation into **Huginn (Thought)** and **Muninn (Memory)**. Huginn (left) runs the live model; Muninn (right) concurrently runs a second provider (default: local Ollama) viewing the same context but with a divergent system prompt — "critique, don't agree." Outputs stream side-by-side at ~380ms offset. Each raven is rendered as a 14px SVG glyph at the top of its pane. When one raven flags a contradiction, its glyph tilts 6°, turns gold for 800ms, and drops a connecting hairline between the two panes at the diverging line.

### 4.3 Sealed Scroll Proof Bundle

On task completion, the proof bundle materializes as a **vellum-textured card** that unrolls from the bottom of the conversation (height 0→auto, 420ms). Four seals in a horizontal row — Tests, Typecheck, Diff, Screenshots — each a 32×32 wax-seal SVG with state: empty circle (pending), spinning rune (running), solid gold (passed), cracked red (failed). Rolled scroll icon at left edge; clicking triggers re-roll animation (scrollY 0→-8px→0 over 320ms) that exports the bundle as a single Markdown file with embedded SHA.

### 4.4 Capability Chips with Provenance Glyphs

Every assistant message header displays a chip strip showing the exact stack that produced it:
- Provider chip: `[Opus ◈ 1M]` gold stroke for paid, green for local
- Augmentation chips: `[🜂 vision]`, `[🜃 thinking]`, `[🜄 tools]` using alchemical sigils
- Cost chip: `$0.08` or `$0 (Ollama)` in hearthgold
- Shadow-git chip: `[⚒ 7a3f2]` truncated SHA with scrubber affordance

### 4.5 The Raven's Flight (iOS↔desktop sync)

When a session transfers device, two raven silhouettes fly across the full-screen canvas on a parabolic arc (800ms, `cubic-bezier(0.25, 0.1, 0.25, 1)`), disappearing through a gold portal in the top-right corner. Opt-in delight.

### 4.6 Sigil Stamp (shadow-git auto-commit)

Before every tool call, shadow-git stamps a checkpoint. A 4px×4px gold sigil blinks once at the top-left of the modified file tab, then contracts into the tab icon. The sigil persists as a subtle 1px gold underline under the filename — hover reveals "12 shadow-commits in this session → scrub timeline."

### 4.7 Council Mode (⌘⇧C)

A single query fans out to 3 or 4 concurrent providers. The conversation pane transforms into a horizontal council table: N columns, each titled with the provider's chip. Responses stream simultaneously. At the bottom, a unified voting bar shows convergence. Operator selects which to merge with `1–4` number keys or drag-up gesture. Dismissed responses fade to 32% opacity but remain in the shadow-git log.

## 5. Three layout innovations (with ASCII wireframes)

### 5.1 The Mead-Hall — Morphing Command Palette

`⌘K` opens a palette centered 38% down the viewport, 640px wide. Three-tier depth by keystroke rhythm:

```
 ┌────────────────────────────────────────────────┐
 │ ᚨ  What would you have me do?                 │
 │ ────────────────────────────────────────────── │
 │  › /plan migrate auth to PASETO               │
 │  › /council review pricing model              │
 │  › /runestone explain shadow-git              │
 └────────────────────────────────────────────────┘
```

Press `⇥` — palette morphs vertically into a planning canvas (height 320→640px, 420ms spring):

```
 ┌────────────────────────────────────────────────┐
 │ ᚨ  migrate auth to PASETO                     │
 │ PLAN                                    REVIEW │
 │ ┌─────────────┐                    ┌─────────┐ │
 │ │ 1. audit    │                    │ critic: │ │
 │ │ 2. draft    │   ← drag to reorder│ missing │ │
 │ │ 3. migrate  │                    │ rotation│ │
 │ │ 4. verify   │                    │ plan    │ │
 │ └─────────────┘                    └─────────┘ │
 │ [ forge ⚒ ] [ summon council ] [ dispatch ]   │
 └────────────────────────────────────────────────┘
```

Press `⇥` again — palette expands to full canvas mode, overlaying the editor. Plan becomes a draggable Miro-like surface with task nodes, agent avatars, file-blast-radius halos. Press `esc` at any tier to collapse one level.

### 5.2 Conversation Braids — Multi-Thread Canvas

`⌘⇧B` opens braid view: 2–4 parallel threads on one canvas, sharing a common context header. Each braid has its own model, tab focus, and goal. Threads can be twisted (merge last N turns of thread B into thread A, with `⟲ braided from muninn-thread-2` provenance).

```
 ┌──── Context: apps/api/src/auth ─────────────────────────────┐
 │ Shared tools: ripgrep, serena, context7                     │
 ├─────────────────────┬──────────────────────┬────────────────┤
 │ ⟐ Opus — architect  │ ⟐ Sonnet — scribe   │ ⟐ Gemma — local│
 │ User: draft a spec  │ User: write JSDoc    │ User: TS skel  │
 │ Opus: ...           │ Sonnet: ...          │ Gemma: ...     │
 │ [twist into main ⤴] │ [twist into main ⤴] │ [retire ×]     │
 └─────────────────────┴──────────────────────┴────────────────┘
```

Cost tracked per-braid in header: `Σ $0.18 · Opus $0.12 · Sonnet $0.06 · Gemma $0.00`.

### 5.3 The Well — Shadow-Git Timeline Scrubber

Pinned to the bottom 48px of the editor (toggleable via `⌘⇧T`). Horizontal ribbon of vertical rune-ticks, one per shadow-commit, colored by event type:

```
 ┌──────────────────────────────────────────────────────────────┐
 │ ᚱ  ᚲ ᚷ    ᛉ ᛉ ᛉ    ᚨ       ᚠ ᚠ    ◆    ᛗ  ᛒ              │
 │ └─────────────────────[ ▼ now ]──────────────────────────────│
 │  9:02am  9:07  9:14    9:21              9:34           9:48 │
 └──────────────────────────────────────────────────────────────┘
```

Scrub `▼` left → editor and chat rewind. Release → "rewind to here?" prompt with Restore (forks branch) vs Inspect only (read-only overlay). Every state always reachable.

## 6. Ten micro-interactions (exact parameters)

| # | Interaction | Parameters |
|---|---|---|
| 1 | **Streaming text fade-in** | Each word enters at opacity 0→1 over 140ms, `cubic-bezier(0, 0, 0.4, 1)`, staggered by 18ms per word. On punctuation insert 60ms micro-pause. Never character-by-character. |
| 2 | **Thinking indicator (rune-forge)** | Three Elder Futhark glyphs (ᚠ ᛉ ᚹ) stroke-dash from 0 to 100% in sequence, 320ms each, then fill with `accent.rune` for 120ms, then un-stroke in reverse. Infinite loop. SVG `stroke-dasharray` + `stroke-dashoffset` animated. Replaces the ubiquitous three-dot spinner. |
| 3 | **Tool call expand** | Collapsed state: 28px pill with tool glyph + name. Click → height 28→auto spring (stiffness 380, damping 32), width 240→100%, content cross-fades at 180ms offset. Re-collapse: reverse with 60ms faster damping. |
| 4 | **Diff accept (physics)** | Accepted hunks slide right 16px then settle (elastic 380ms), green-flash background `rgba(79,163,127,0.3)` → transparent over 620ms. Rejected: slide left 24px, fade 240ms, collapse height 180ms — removed from DOM. |
| 5 | **Provider chip hover** | `transform: scale(1.04)`, elevation 1→2 over 180ms `motion.glide`. Chip text slides up 2px; 1px accent border grows from center to full width in 220ms. |
| 6 | **Message hover actions** | Right-side 24px ghost icons (copy, branch, save-as-skill, star). Fade in 0→0.7 over 160ms, stagger 24ms each. Individual icon hover: opacity 1 + 2px lift. |
| 7 | **Input focus ritual** | Border 1px `surface.stroke` → 2px `accent.rune` over 180ms; simultaneously 1px inner glow breathes at 2s `sin` (opacity 0.1↔0.25). On Enter: border flashes `accent.gold` 140ms before message lifts out. |
| 8 | **Sidebar collapse** | Width 260px → 56px over 320ms `motion.glide`. Labels cross-fade 0→120ms; icons remain. On re-expand: spring + labels stagger-in 24ms from top. |
| 9 | **Error shake** | `translateX(0, -4px, 4px, -2px, 2px, 0)` over 320ms, `cubic-bezier(0.36, 0.07, 0.19, 0.97)`. Only for destructive confirmations. Never on parse errors. |
| 10 | **Session end — Ember** | On app quit: all text fades to 40% opacity (640ms), then small ember particle emerges at center, burns out over 880ms (radius 6→0, opacity 1→0, slight upward drift y 0→-24px). Then hard cut. Users remember this. |

## 7. Five onboarding hooks

### 7.1 The Well of Mimir (recommended primary)

First-run opens a full-screen Runestone theme viewport. Centered: a still pool of water rendered as a 480px circle with a 14-frame subtle caustic shader (perlin noise). Text: *"Drink from the well. What do you seek?"* The textarea is the pool's reflection — typing animates ripples outward. On submit, the water drains downward (800ms) revealing the full 3-pane app, with the user's query already being answered. **First-use-to-first-answer target: under 8 seconds.**

### 7.2 The Summoning (choose your patron)

Four cards fan out horizontally, each representing a work style:
- **Thor** (fast, brute-force): cheap models, parallel by default, risk-tolerant
- **Odin** (wise, deliberative): premium models, plan-first, verify-heavy
- **Loki** (creative, disruptive): council mode on, divergent prompts
- **Freya** (nurturing, pair-programming): test-first, small-step, copilot-style

Each card previews its config as a 4-line cost+behavior summary. Pick one → immediately usable settings preset.

### 7.3 The Rune Revealed (keyboard shortcut tutor)

60-second interactive tour: empty editor with a single floating rune. Caption: *"Press ⌘K. That is Ansuz, the messenger."* Six runes total, each mapped to a shortcut. At the end: *"You know the runes. The hall is yours."* Skippable with `esc`. For users who complete it, a small rune-row persists in the status bar.

### 7.4 Cost Glint (the moment a paid key is added)

Initial state: cost indicator reads **"$0.00 (Ollama local)"** in moss-green. When user pastes an API key in Settings, the indicator glints gold — 4px specular sheen sweeps over 720ms, and a small scroll unfurls below: *"Hearthgold unlocked. Opus available. Local models still preferred when sufficient."* Reinforces that WOTANN defaults to free.

### 7.5 First Quest

After onboarding, the left sidebar shows a single card: **"Your first quest: complete any task end-to-end and earn your first proof-seal."** When the user completes anything with a proof bundle, the card flips with a satin-fold animation (720ms) to reveal the seal.

## 8. Sound design (opt-in)

All sounds produced in-house (WAV, 48kHz, 24-bit, peak -12dBFS). All <400ms. Informed by physical craft, not UI-kit stock.

| Cue | Duration | Character | Trigger |
|---|---|---|---|
| **Rune-tap** | 80ms | Wooden mallet on bronze, 320Hz + 1.2kHz overtone | Tool-call success |
| **Well-hum** | 2400ms loop, -22dB | Low cavern tone, 80Hz w/ 0.3Hz wobble | Session open (first 2.4s) |
| **Wax-seal press** | 180ms | Low-mid thud + brief high sizzle tail | Proof bundle sealed |
| **Wood-knock** | 120ms | Single oak rap, no resonance | Error shake / destructive warning |

**Never**: beeps, swooshes, button clicks, success chimes. Default: all off.

## 9. Mobile / Watch / Widgets / CarPlay

### 9.1 iOS App (3-tab translation)

- **HomeView** — vertical feed of *conversation spines* (each spine = color ribbon by active patron, thickness = activity level, height = recency). Tap spine to expand. Pull-down = `⌘K` equivalent as glass sheet.
- **WorkView** — swipeable diff cards (left = reject, right = accept — Tinder-style with haptic detents at ±18°).
- **ArenaView** — Exploit/red-team tab. Tab bar gains blood-accent when red-team job running; tap-rigid-heavy haptic on critical findings.

### 9.2 Apple Watch

- **Complication** (circular): single rune by state — Fehu (idle), Raidho (running), Ansuz (awaiting input), Tiwaz (proof-seal complete)
- **Glance**: 3 lines — patron sigil + cost today, "Huginn is scouting…" or "Complete" + rune, tap-to-talk button

### 9.3 Home Screen Widgets

- **Small (2×2)**: one big rune + one metric. Rune pulses when autonomous agent needs input.
- **Medium (4×2)**: rune + current quest title + "X of Y steps" + mini-progress ring
- **Large (4×4)**: braid view: 3 stacked ribbons showing last assistant line truncated, each tappable. Bottom: "Summon" button → voice capture.

### 9.4 Live Activities / Dynamic Island

- **Compact leading**: spinning rune (Raidho)
- **Compact trailing**: cost ticking, hearthgold for paid, moss-green for local
- **Expanded**: current message first 40 chars; progress bar; "Leash" button (stops agent). Copy by tool: `reading`, `searching`, `forging`, `sealing`.

### 9.5 CarPlay Templates

- **`CPListTemplate`** — one row per active thread, patron color ribbon. Tap → Siri reads assistant output in thread's voice (Odin = deeper, Freya = warmer).
- **`CPGridTemplate`** — five tiles: Summon, Status, Cost, Logs, Mute.
- **Safety**: tool execution SUSPENDED while driving. Only read/plan operations run. Spoken notice: *"The forge rests while you drive."*

## 10. Accessibility floor

- Minimum contrast 7:1 for all primary ink-on-background pairs (Mimir, Runestone, Valkyrie pass; Yggdrasil at 10:1)
- All rune animations respect `prefers-reduced-motion: reduce` — reduced to 2-state opacity fade over 120ms, no transforms
- Every rune has `aria-label` with name and meaning
- Minimum tap target 44×44pt on touch; 32×32pt on desktop with expanded pointer-events padding
- Screen-reader mode: rune-glyph indicators become text status; wax-seal animations become "Sealed" announcements
- Colorblind-safe palettes for Valkyrie (red/green distinguishable by **glyph shape** — cracked vs solid seal — not just color)
- Focus rings non-negotiable: `outline: 2px solid #FFA843; outline-offset: 2px` on every focusable element (competitors universally fail this — free differentiator)

## 11. Anti-patterns (what WOTANN refuses)

To stay distinctive, WOTANN **does not do** the things every competitor does:

1. **No purple**. Not a single shade, anywhere, ever.
2. **No "AI sparkle" icon** — Ansuz (ᚨ) is the messenger glyph.
3. **No chat bubble rounded-rectangle messages** — messages are flush-left typographic blocks with left-hairline accent.
4. **No gradient avatar rings**. No animated "thinking" avatars. Providers are named, shown as flat chips.
5. **No "This is experimental" disclaimers** on stable features. Either ship or hide.
6. **No tooltips for discoverable features**. If you need a tooltip to explain it, it isn't designed right.
7. **No emoji in system UI**. Runes, sigils, alchemical marks — all custom SVG, in the type system.
8. **No ghost onboarding text** ("Try asking me about…"). The Well of Mimir asks a real question.
9. **No right-rail collapsed "agent scratchpad"** — agent thinking is in the conversation or in a dedicated scratchpad panel, never a mystery sidebar.
10. **No notifications badge counts without meaning**. Every badge has a reason a user can read on hover.

## 12. Competitor-specific patterns worth porting

From the 33-project competitor research synthesized in session-5/6:

### P10 — Ship first

1. **Magika file-type gate** (google/magika, 0 LOC port) — content-aware routing; disguised-binary detection → Exploit tab. **LANDED session-6 commit af46533.**
2. **Block UI primitive** (Warp, ~600 LOC) — each cmd+output = discrete selectable/shareable/rerunnable unit with rule dividers + context menu. Map to WOTANN's message turns.
3. **Symbol-level @ mentions** (Air, ~450 LOC) — class/method/line/commit as first-class mention types beyond @file.
4. **Multibuffer** (Zed, ~900 LOC) — single editable view across search hits / diagnostics / agent edits. Deeper than WOTANN's MultiFileComposer.
5. **Agent Skills open-standard compliance** (OpenAI+Hermes+Superpowers, ~350 LOC) — adopt `agentskills.io` SKILL.md frontmatter. Instant 1000s-of-skills library.
6. **`[POINT:file:line]` protocol + bezier cursor overlay** (Clicky, ~600 LOC) — agent physically gestures at code. **Unique — nobody else has it.**
7. **Headless + mobile UI** (Jean, ~700 LOC) — Tauri serve-mode + mobile PWA with token auth. Access from phone via Tailscale/Cloudflare Tunnel.

### P9 — Ship second

8. **Focus View** (Claude Code `/focus`, ~300 LOC) — 3-line collapse: last-prompt / 1-line tool summary w/ diffstats / final response.
9. **Credential pool + rate guard + prompt caching** (Hermes, ~600 LOC) — OAuth rotation, predictive throttling, cross-provider caching.
10. **Conductor's "reset to previous turn"** (~200 LOC on existing shadow-git) — expose per-turn rewind as `/undo-turn` slash + sidebar UI.
11. **@terminal chat mention** (Conductor, ~180 LOC) — attach live terminal output into prompt composer as typed mention.
12. **Context-meter radial dial** (Conductor, ~280 LOC) — hover-reveal breakdown of context usage by category.
13. **Agent Profiles cycled via Shift+Tab** (Zed + Air, ~350 LOC) — packaged permission presets with 1-key cycle.
14. **Execution Environments radio per task** (Air, ~550 LOC) — Local / Worktree / Docker per-task with permission-mode quartet.
15. **Per-prompt model + effort override + `xhigh`** (Jean, ~220 LOC) — change model/effort PER MESSAGE, not per session.
16. **Karpathy-mode system-prompt toggle** (4 principles, 80 LOC rewrite) — **LANDED session-5 commit 41be4de.**

### P8 — Ship third
