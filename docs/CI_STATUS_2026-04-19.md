## Fix (Gabriel action required — ~4 min)

From `docs/SELF_HOSTED_RUNNER_SETUP.md` verbatim, with the two substitutions below.

### Step A: Get a fresh registration token

Open https://github.com/gabrielvuksani/wotann/settings/actions/runners/new, click "macOS" / "ARM64", and copy the `--token XXXX` value. Tokens expire in 1 hour.

### Step B: Register and start (macOS arm64 host)

```bash
# 1. Create a runner work directory (anywhere outside ~/Desktop/agent-harness/)
mkdir -p ~/actions-runner && cd ~/actions-runner

# 2. Download the latest runner for macOS arm64
#    (check the URL on the GitHub settings page; version may have bumped
#    past 2.321.0 by the time you read this — use the download URL
#    GitHub shows you)
curl -o actions-runner-osx-arm64.tar.gz -L \
  https://github.com/actions/runner/releases/download/v2.321.0/actions-runner-osx-arm64-2.321.0.tar.gz
tar xzf actions-runner-osx-arm64.tar.gz

# 3. Register with the repo — using labels that match ci.yml current state
./config.sh --url https://github.com/gabrielvuksani/wotann --token <TOKEN_FROM_STEP_A> \
  --labels self-hosted,linux --unattended --replace

# 4. Start the runner in the foreground (Ctrl-C to stop; use ./svc.sh for a
#    launchd service instead — see doc for the persistent-service route).
./run.sh
```

Once `./run.sh` prints `Connected to GitHub`, the queued run `24631396542` should be picked up within ~30s. Verify with:

```bash

```
