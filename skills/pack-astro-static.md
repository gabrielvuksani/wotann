# Pack: Astro (static)

A minimal but functional Astro 4 scaffold emitted by `wotann build`
when the selected scaffold is `astro-static`. After materialization the
project boots with `npm install && npm run dev` and produces a static
site via `npm run build`. Zero client JavaScript is shipped by default —
add `client:*` directives only where needed.

### README.md

```md
# Wotann App — Astro static

Generated by `wotann build` using the **astro-static** scaffold.

## Run locally

```bash
npm install
npm run dev
```

Open http://localhost:4321. Astro hot-reloads on save.

## Build

```bash
npm run build
npm run preview
```

The static output lands in `dist/` — drop it on any CDN (Cloudflare
Pages, Netlify, Vercel static, GitHub Pages, S3+CloudFront, …).

## What's inside

- `src/pages/index.astro` — landing page (file-based routing)
- `src/layouts/Base.astro` — shared layout with global styles
- `astro.config.mjs` — static output, directory format
- `tsconfig.json` — extends Astro's strict preset

## Add later

- `@astrojs/mdx` — Markdown + JSX content
- `@astrojs/sitemap` — automatic sitemap.xml
- `@astrojs/cloudflare` / `@astrojs/vercel` — switch to SSR

## Provenance

Materialized from `skills/pack-astro-static.md` by Wotann's build
pipeline. See `docs/build-templates/README.md` for the full flow.
```

## Provenance

WOTANN scaffold pack used by `wotann build` when the registry picks
`astro-static`. Tested with Astro 4.13.x and TypeScript 5.5.x. The
file list is kept in 1:1 sync with `SCAFFOLDS[astro-static].files` in
`src/build/scaffold-registry.ts`.
